📊 If-Else Statement (Penentuan Grade)

Nilai: 85 → Grade: B (Baik)
Nilai: 55 → Grade: E (Sangat Kurang)

Alasan Pemilihan If-Else:

  • Range Berurutan: Nilai memiliki range yang berurutan (90-100, 80-89, dll)
  • Kondisi Bertingkat: If-Else ideal untuk kondisi yang saling berkaitan
  • Evaluasi Berurutan: Pengecekan dilakukan dari nilai tertinggi ke terendah
  • Default Case: Else menangani semua nilai di bawah 60

🔐 If-Else Statement (Validasi Input)

Username: "admin", Password: "12345" → Login Berhasil
Username: "user", Password: "wrong" → Password salah!
Username: "", Password: "12345" → Username tidak boleh kosong!

Alasan Pemilihan If-Else untuk Validasi:

  • Validasi Bertahap: If-Else ideal untuk pengecekan step-by-step
  • Kondisi Kompleks: Dapat menangani nested conditions dengan baik
  • Early Return: Bisa langsung return ketika validasi gagal
  • Readability: Mudah dibaca dan dipahami flow validasinya
  • Flexibility: Bisa menambahkan kondisi validasi baru dengan mudah

🔥 Kombinasi If-Else & Switch-Case

Kategori B + Skor 85 → Kategori B - Good Job!
Kategori C + Skor 75 → Perlu improvement

Alasan Kombinasi:

  • Kondisi Bertingkat: If-Else untuk range skor, Switch untuk kategori
  • Efisiensi: Switch hanya dieksekusi jika skor >= 80
  • Flexibility: Kombinasi memberikan fleksibilitas untuk logika kompleks
  • Maintainability: Mudah ditambah kategori baru tanpa mengubah struktur
